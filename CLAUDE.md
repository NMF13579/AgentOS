## 4. Правила работы агента

### Перед любой задачей:
1. Прочитать `HANDOFF.md` → `memory-bank/PROJECT-MEMORY.md` → `memory-bank/project-status.md` → `docs/ROADMAP.md` (если файл существует)
2. Показать план → получить подтверждение
3. Только после «да» — писать код

### После каждой задачи:
1. Обновить `HANDOFF.md`
2. Обновить `memory-bank/project-status.md`
3. Добавить запись в `memory-bank/features.md` или `fixes.md`

---

## 5. Когда агент обязан остановиться

- Задача затрагивает данные пользователей или auth
- Нужно удалить, переименовать таблицу или поле в БД
- Предлагается новая внешняя зависимость
- Задача выходит за рамки текущего этапа ROADMAP
- Непонятно что именно хочет владелец
- Владелец просит добавить новую функцию в середине задачи → прочитать `docs/SCOPE-CREEP-GUARD.md`

**Формула:** «Стоп. Прежде чем продолжить — мне нужно уточнить: [вопрос].»

## 8. Автовыбор режима (AUTO MODE SELECTION)

Агент определяет MODE автоматически на основе контекста сессии:

| Ситуация | MODE | Что делает агент |
|----------|------|-----------------|
| Задача размытая, цель неясна | `EXPLORATION` | Задаёт уточняющие вопросы, не пишет код |
| Цель ясна, плана нет | `PLANNING` | Предлагает план и ждёт подтверждения |
| План подтверждён | `EXECUTION` | Пишет код строго по плану |
| Владелец совершает повторяющиеся ошибки | `STRICT` | Добавляет явные предупреждения перед каждым шагом |
| Нужно быстро, детали не важны | `FAST_TRACK` | Минимум вопросов, максимум скорости |

**Правило:** агент называет текущий MODE в начале каждой сессии:
> «Режим: PLANNING. Вижу что план ещё не подтверждён — давай зафиксируем шаги.»

Владелец может изменить MODE командой (см. раздел 9).

## 9. Команды владельца (COMMAND LAYER)

Агент обязан реагировать на эти фразы немедленно:

| Команда | Действие агента |
|---------|----------------|
| «дай статус» | Показать: текущий MODE, активная задача, STATE, следующий шаг |
| «что дальше» | Показать следующий шаг по ROADMAP |
| «сделай быстрее» | Переключить MODE = FAST_TRACK, предупредить о рисках |
| «сделай нормально» | Переключить MODE = STRICT, пройти чеклист заново |
| «проверь» | Запустить `docs/TASK-REVIEW-PROTOCOL.md`, затем `docs/VALIDATION.md` |
| «стоп» | Остановиться, показать что было сделано, ждать команды |
| «откат» | Запустить `docs/ROLLBACK-PROTOCOL.md` |
| «что сломалось» | Запустить `docs/TESTING-GUIDE.md` |
| «восстанови контекст» | Запустить `docs/CONTEXT-LOSS-RECOVERY.md` |

> Команды работают в любой момент сессии, в любом MODE.
