# ADVANCED-SETUP — техническое приложение

> Для продвинутой настройки среды, CLI и интеграций.  
> Базовый запуск без терминала: `QUICK-START.md`.

---

## 1) Настройка `.claude/` и `.cursor/`

### `.claude/`

- Используйте `CLAUDE.md` как главный свод правил поведения агента.
- Для этапной работы подключайте `stages/0X-*/BOOT.md`.
- Для общих правил и ограничений используйте `shared/`.

### `.cursor/`

- Правила автоприменения держите в `.cursor/rules/`.
- Ключевые файлы правил:
  - `00-core.mdc` — базовый цикл сессии
  - `10-communication.mdc` — стиль общения
  - `30/31/32/33` и `40/50/60` — маршрутизация и приоритеты
- Для одинакового поведения в Cursor и Claude поддерживайте единый источник правил в документах `LAYER-1/`.

---

## 2) Выбор LLM и режим работы

- Базовый режим: модель по умолчанию среды (Cursor/Claude Code).
- Для сложных задач (архитектура, аудит, риски) используйте более точную модель.
- Для быстрых итераций и рутинных правок используйте fast-режим.
- Всегда проверяйте, что финальные изменения проходят self-verification и согласование с владельцем.

Практика выбора:

1. **Планирование / комплаенс / безопасность** → приоритет качеству ответа.
2. **Массовые мелкие правки** → приоритет скорости.
3. **Критичные решения** → повторная проверка на той же или более сильной модели.

## 4) Конфигурация MCP

В проекте доступны MCP-серверы для браузера, Vercel, Supabase, Figma, Appwrite.

Рекомендуемый порядок подключения:

1. Проверить доступные инструменты сервера через его descriptor.
2. Подтвердить аутентификацию (если сервер требует `mcp_auth`).
3. Ограничить права по принципу least privilege.
4. Для прод-среды не использовать тестовые ключи и наоборот.

Базовые правила:

- Не передавать секреты в явном виде в чат.
- Не выполнять деструктивные команды без явного подтверждения.
- Логи и артефакты с ПДн хранить в контролируемой среде.

## 5) Когда использовать это приложение

- Нужна глубокая настройка агента под проект.
- Нужно подключить или отладить MCP-интеграции.
- Нужно стандартизировать процесс для команды разработки.

Если цель — быстро начать без технической настройки, вернитесь в `QUICK-START.md`.
