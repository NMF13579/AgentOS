## Distilled Lesson

M50 exists because a validated proposal can be wrongly interpreted as immediate permission to execute.
Validated proposal is not permission to execute.
Conversion authorization is not execution authorization.
Task contract candidate is not active task state.
M50 creates a candidate for later placement review, not an executable lifecycle state.

## Why This Boundary Exists

Без этой границы легко смешать проверку предложения и право на запуск работы.
M50 отделяет этап подготовки кандидата от этапа жизненного цикла, где возможны размещение и исполнение.
Это снижает риск несанкционированного запуска, тихого расширения объема и потери ограничений.

## Core Authority Separation

- proposal ≠ task contract
- proposal validation ≠ execution permission
- conversion ≠ execution
- human authorization for conversion ≠ execution approval
- validator PASS ≠ approval
- task contract candidate ≠ active task
- task contract candidate ≠ queue entry

## High-Risk Failure Patterns

- validated proposal treated as executable task
- validator PASS treated as approval
- generic approval treated as scoped authorization
- conversion authorization treated as execution approval
- task contract candidate copied into tasks/active-task.md
- task contract candidate placed into tasks/queue/
- mode EXECUTION used instead of EXECUTION_SHAPE
- execution_authorized set to true
- execution_permission_granted set to true
- active_task_allowed set to true
- task_queue_allowed set to true
- accepted limitations dropped
- open questions dropped
- downstream limits dropped
- forbidden changes weakened
- allowed changes expanded
- source traceability missing
- approval record generated by validator

## Human Authorization Lesson

Human authorization must be explicit.
Human authorization must be scoped.
Human authorization must reference a specific proposal.
Generic approval is not sufficient.
M49_COMPLETE is not human authorization.
Human authorization must be valid when the conversion validator runs.
Human authorization for conversion does not authorize execution.
Human authorization for conversion does not authorize queue placement.
Human authorization for conversion does not authorize active-task replacement.
Human authorization for conversion does not authorize implementation.

Starting conversion before authorization expiration is not sufficient.
Validator-time validity is required.

## Validator Lesson

Conversion validator checks whether conversion is valid.
Conversion validator does not authorize execution.
Conversion validator does not place task into queue.
Conversion validator does not create active task state.
Conversion validator does not create approval records.
Conversion validator PASS is not approval.

## EXECUTION_SHAPE Lesson

M50 uses EXECUTION_SHAPE, not EXECUTION.
EXECUTION_SHAPE does not mean execution is authorized.
EXECUTION_SHAPE does not allow queue placement.
EXECUTION_SHAPE does not allow active-task replacement.
EXECUTION_SHAPE does not allow implementation.

EXECUTION_SHAPE means structural task-contract shape only.
