# GATE-RESULT-SEMANTICS

## Purpose
Правила трактовки итоговых статусов gate-проверок.

## Meaning
- PASS: все обязательные проверки пройдены
- WARN: обязательное прошло, но есть предупреждения
- FAIL: есть обязательная непройденная проверка
- ERROR: внутренняя ошибка проверки

## Additional Statuses
Статусы отдельных пунктов проверки:
- NOT_RUN: проверка не запускалась
- NOT_IMPLEMENTED: проверка ещё не реализована

NOT_RUN и NOT_IMPLEMENTED не равны PASS.

## Safety Boundary
Даже PASS не отменяет ручное решение владельца о готовности к выпуску.
