# UX-CHECKLIST-LEGEND.md

Легенда и правила использования UX-чек-листов.

## 1. Обозначения

Во всех UX-чек-листах используются одинаковые отметки:

- `[x]` — включено по умолчанию  
  Элемент считается стандартом для данного чек-листа.  
  Агент может использовать его без дополнительного одобрения, если чек-лист применён.

- `[!]` — обязательно  
  Критичный элемент, без которого сценарий считается небезопасным или некорректным.  
  Агент обязан его учесть, если чек-лист применён (особенно в медицинских и высокорискованных сценариях).

- `[ ]` — опционально  
  Дополнительный элемент. Агент может предложить его, но не обязан включать по умолчанию.  
  Включение зависит от контекста проекта и согласования с пользователем.

## 2. Как агент выбирает чек-листы под проект

Шаги:

1. Определить тип проекта:
   - Простой кабинет / MVP
   - Внутренний сервис / админка / B2B
   - Медицинский или около-медицинский сервис
   - Публичный личный кабинет / портал
   - Сильный мобильный сценарий (mobile-first)

2. По `UX-CHECKLIST-INDEX.md` выбрать набор чек-листов:
   - Всегда: базовые (`DEFAULT`, `ACCESSIBILITY`, `EMPTY-STATES`), если не сказано иное.
   - Добавлять доменные (`MEDICAL`, `ROLE-BASED-UX`, `DASHBOARDS`) по типу проекта.
   - Добавлять специализированные (`NOTIFICATIONS`, `SEARCH-FILTERS`, `TABLES-AND-BULK-ACTIONS`, `HISTORY-AND-AUDIT`, `MOBILE`, `HELP-AND-SUPPORT`, `MICROCOPY`) по реальной потребности.

3. Сформулировать предложение пользователю:

   Пример:
   > "Для этого проекта предлагаю использовать чек-листы: DEFAULT, ACCESSIBILITY, EMPTY-STATES, MEDICAL, MOBILE и NOTIFICATIONS. Это даст стандартную структуру экранов, понятные empty states, медицинскую безопасную логику и mobile-first UX. Подтверждаете такой набор?"

4. После согласия:
   - Агент использует все пункты `[!]` из выбранных чек-листов как обязательные.
   - Агент использует пункты `[x]` как стандартный набор.
   - Пункты `[ ]` агент предлагает как улучшения по мере необходимости.

## 3. Приоритеты

Если возникает конфликт:

- Между “красотой” и доступностью → приоритет у доступности.
- Между “минимализмом” и безопасностью → приоритет у безопасности (особенно в медицине).
- Между скоростью внедрения и ясностью интерфейса → приоритет у ясности для пользователя.

## 4. Рекомендация по применению в медицине

Для медицинских и около-медицинских проектов агент должен:

- Всегда включать:
  - `UX-CHECKLIST-MEDICAL.md`
  - `UX-CHECKLIST-ACCESSIBILITY.md`
  - `UX-CHECKLIST-EMPTY-STATES.md`
  - `UX-CHECKLIST-NOTIFICATIONS.md`
  - `UX-CHECKLIST-HISTORY-AND-AUDIT.md`
- Для ролевых систем:
  - Добавлять `UX-CHECKLIST-ROLE-BASED-UX.md`
- Для дашбордов:
  - Добавлять `UX-CHECKLIST-DASHBOARDS.md`
- Для мобильных сценариев:
  - Добавлять `UX-CHECKLIST-MOBILE.md`

Агент должен явно фиксировать в начале работы:  
“Этот проект использует следующие UX-чек-листы: …” и придерживаться их при проектировании интерфейсов.
