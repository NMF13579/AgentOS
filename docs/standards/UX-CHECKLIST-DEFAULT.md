# UX-CHECKLIST-DEFAULT.md

Чек-лист стандартных UX-решений для агента.
Агент использует отмеченные пункты как базовый интерфейс после согласия пользователя.
Пункты `[x]` включаются по умолчанию.
Пункты `[ ]` считаются опциональными.
Пункты `[!]` обязательны для медицинских интерфейсов.

## 1. Базовая структура экрана

- [x] Заголовок страницы.
- [x] Короткое описание назначения экрана.

- [x] Один основной CTA на экран.
- [x] Вторичные действия визуально слабее главного.
- [ ] Блок помощи или подсказок.
- [ ] Блок "что делать дальше".

## 13. Медицинский UX

- [!] Главные данные выше второстепенных.
- [!] Максимально ясные названия действий.
- [!] Подтверждение чувствительных действий.
- [!] Минимум неоднозначных формулировок.
- [!] Ошибки должны быть безопасными и понятными.
- [!] Важные статусы выделены ясно, но не только цветом.
- [ ] История действий пользователя.
- [ ] Предупреждения для чувствительных данных.
- [ ] Режим повышенной читаемости.
- [ ] Пошаговые сценарии для критичных процессов.

## 14. Правило применения агентом

- Агент использует все пункты `[x]` как стандартный UX-набор.
- Агент использует все пункты `[!]` как обязательные для медицинских интерфейсов.
- Пункты `[ ]` агент предлагает отдельно как улучшения.
- Если пользователь просит минимальный интерфейс, агент оставляет базу из `[x]` и `[!]`.
- Если пользователь просит расширенный интерфейс, агент может добавить часть пунктов `[ ]`.

## 15. Фраза для согласования с пользователем

"Могу собрать интерфейс по стандартному UX-чек-листу: базовая структура экранов, состояния загрузки и ошибок, формы, таблицы, подтверждения опасных действий, мобильная адаптация и медицинские требования, если проект связан с медициной. Необязательные элементы можно отключить. Применяем?"
