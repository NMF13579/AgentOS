# UX-CHECKLIST-SEARCH-FILTERS.md

Чек-лист поиска и фильтров.
Цель — сделать поиск и фильтрацию данных понятными и предсказуемыми.
`[x]` — по умолчанию, `[ ]` — опционально.

## 1. Поиск

- [x] Поиск там, где пользователь ожидает его увидеть.
- [x] Понятное поле с placeholder (что можно искать).
- [x] Чёткое поведение: поиск по Enter или по вводу.
- [ ] Подсказки по поисковым запросам.
- [ ] Автодополнение.

## 2. Фильтры

- [x] Только реально нужные фильтры, без перегруза.
- [x] Понятные названия фильтров.
- [x] Видно, какие фильтры сейчас активны.
- [ ] Панель фильтров, которую можно свернуть.
- [ ] Сохранённые наборы фильтров.

## 3. Сортировка

- [x] Сортировка по 1–2 главным колонкам.
- [x] Понятно, по какому полю сейчас идёт сортировка.
- [ ] Возможность менять направление сортировки.
- [ ] Сохранение последней сортировки для пользователя.

## 4. Сброс параметров

- [!] Есть простой способ сбросить фильтры и поиск.
- [x] Вариант “сбросить всё” не прячется.
- [ ] Кнопка “вернуться к рекомендациям по умолчанию”.

## 5. Empty state с фильтрами

- [x] Если ничего не найдено — есть текст “ничего не найдено”.
- [x] Есть подсказка “измените фильтры или запрос”.
- [x] Есть кнопка “сбросить фильтры” или аналог.
- [ ] Примеры запросов, которые дадут результаты.

## 6. Мобильный контекст

- [x] Фильтры не занимают весь экран постоянно.
- [x] Есть кнопка открытия/закрытия фильтров.
- [ ] Sticky кнопка “фильтры”/“поиск”.
- [ ] Bottom sheet для фильтров.

## 7. Медицинский контекст

- [x] Фильтры и поиск не должны приводить к “пропаже” пациентов/документов без понятного объяснения.
- [x] При критичных списках (пациенты, случаи, результаты) должно быть понятно, что список ограничен фильтрами.
- [ ] Отдельные фильтры для приоритетных статусов (например, “критичные результаты”).
- [ ] Явная подсветка активных фильтров в таких списках.

## 8. Правило применения

- Агент включает минимальный поиск/фильтры везде, где есть большие списки.
- Расширенные фильтры добавляются только при реальной потребности, а не “на всякий случай”.
