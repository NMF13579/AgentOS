# Plan Gate и защита скоупа — жёсткие правила

---

## Plan Gate (перед любыми действиями с файлами или кодом)

1. Сформулировать план на русском: что делаешь, какие файлы трогаешь, что **не** трогаешь.
2. Показать план владельцу. Ждать явного подтверждения: «да», «ок», «выполняй».
3. Только после подтверждения — исполнять.

**Молчание ≠ согласие.** Без подтверждения плана — не начинать.

**LOCKED-секции:** не редактировать секции с `LOCKED:true` в `project/PROJECT.md` без явного решения владельца.

**Один запрос = одна задача.** Расширение скоупа — отдельное согласование через `shared/scope-guard.md` и `LAYER-1/scope-guard.md`.

---

## Жёсткие правила скоупа (ничего лишнего)

- **One task at a time** — завершить текущую задачу, прежде чем принимать следующую.
- **No silent refactoring** — улучшение вне скоупа: зафиксировать в логе отложенных решений/фич/фиксов, **не** делать молча.
- **No new dependencies** — не добавлять пакеты, библиотеки или сервисы, которых нет в плане.
- **No file creation outside plan** — каждый новый файл должен быть в подтверждённом плане.
- **No config changes** — не трогать `.env`, CI и инфраструктуру без явного подтверждения.

---

## Когда задача оказалась шире плана

Сразу остановиться и сообщить владельцу по шаблону:

```
🚧 СТОП — задача оказалась шире плана

Обнаружил: [что нашёл]
Это затрагивает: [файлы/компоненты]

Варианты:
А) Продолжить только текущий план, остальное — в следующую задачу
Б) Обновить план и подтвердить снова

Что делаем?
```

---

## Куда логировать отложенное (не в чат)

Каждое улучшение или находка **вне текущего скоупа** фиксировать в файлах:

- архитектурный выбор → `LAYER-3/deferred-decisions.md`
- идея фичи → `LAYER-3/features.md`
- известный баг / техдолг → `LAYER-3/fixes.md`

**Не** полагаться на чат как на единственную память.
