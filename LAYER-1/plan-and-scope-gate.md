# Plan Gate и защита скоупа

Свод правил из Cursor `30-plan-gate.mdc` и `60-scope-guard.mdc` (канон в `LAYER-1/`).

---

## Plan Gate

**Перед любым действием** с файлами или кодом:

1. План на русском: что делаешь, какие файлы трогаешь, что **не** трогаешь.
2. Показать план владельцу. Ждать явного «да» / «ок» / «выполняй».
3. Только после подтверждения — исполнять.

**Молчание ≠ согласие.** Без подтверждения плана — не начинать.

**LOCKED:** не редактировать секции с `LOCKED:true` в `project/PROJECT.md` без явного решения владельца.

**Один запрос = одна задача.** Расширение скоупа — отдельное согласование через `shared/scope-guard.md` и `LAYER-1/scope-guard.md`.

---

## Жёсткие правила скоупа

- **One task at a time**
- **No silent refactoring** — вне скоупа: лог в файлы (см. ниже), не делать молча
- **No new dependencies** вне плана
- **No file creation outside plan**
- **No config changes** — `.env`, CI, инфра — только с явным подтверждением

```
Б) Обновить план и подтвердить снова

Что делаем?
```
