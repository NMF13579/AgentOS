# Stage Routing — маршрутизация по этапам

У каждого проекта на этом фреймворке **4 этапа**. Каждую задачу нужно отнести к правильному этапу и открыть соответствующий `BOOT.md`.

## Карта этапов

| Этап | Папка | BOOT.md | Когда использовать |
|------|--------|---------|---------------------|
| 0 — Discovery | `stages/00-discovery/` | `BOOT.md` | Новый проект, цели, интервью |
| 1 — Planning | `stages/01-planning/` | `BOOT.md` | Архитектура, стек, roadmap |
| 2 — Build | `stages/02-build/` | `BOOT.md` | Разработка, фичи |
| 3 — Deploy | `stages/03-deploy/` | `BOOT.md` | CI/CD, релиз, мониторинг |

## Как маршрутизировать

1. Перед началом задачи определить этап.
2. Прочитать `BOOT.md` этапа (например `@stages/00-discovery/BOOT.md`).
3. Следовать чеклисту и ссылкам внутри `BOOT.md`.
4. Если этап неясен — задать **один** вопрос владельцу: «Это этап [N] или [N+1]?»

## Переход между этапами

- Не переходить к следующему этапу, пока не закрыт чеклист текущего.
- При переходе: обновить `HANDOFF.md`, `LAYER-3/project-status.md`, согласовать с владельцем.
- Отложенное → `LAYER-3/deferred-decisions.md`.

## Сквозные отсылки

- Спеки: `LAYER-2/specs/` — проверять перед планированием.
- Решения: `LAYER-3/deferred-decisions.md` — читать перед архитектурными выборами.
- Уроки: `LAYER-3/lessons.md` — читать, чтобы не повторять ошибки.
